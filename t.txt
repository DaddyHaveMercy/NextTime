### 4. Comparison and update logic (mise à jour)

Cette section doit maintenant traiter les deux diagrammes indépendamment. Remplace-la par :

```markdown
### Comparison and update logic

For each of the two diagrams (identified by class suffix
`aidpgen-flowchart` and `aidpgen-sequence`) independently:

1. Generate the current Mermaid text from this AGENTS.md.
2. In the page body returned at Step 4, locate the existing diagram:
   - Find `<ac:structured-macro ac:name="html">` whose CDATA body contains
     `<pre class="mermaid <suffix>">` where `<suffix>` is the one being
     compared.
   - Extract the text **between** the opening `<pre ...>` and the closing
     `</pre>` (ignore the trailing `<script>` if any).
3. Normalize both:
   - Strip leading/trailing whitespace.
   - Collapse any run of whitespace characters into a single space.
4. Compare. If equal → mark this diagram "unchanged". If different (or no
   existing macro found) → mark this diagram "needs update".

After both diagrams have been evaluated:

- If BOTH are unchanged → print `Diagrams unchanged, not republished.` and
  proceed to the next step. Do NOT call the update-page tool.
- If AT LEAST ONE needs update → build the new page body by replacing the
  affected macro(s) in place (leave the unaffected macro exactly as it was
  on the server, to preserve any manual edit the user may have made to the
  section header). If a macro is missing entirely, prepend the missing
  section at the top of the body, respecting the order above.
- Call the `DevopsProdITG` Confluence update-page tool once with the full
  new body.

Confirm with:
`Diagrammes publiés sur Confluence: <list of updated ones>` — e.g.
`flowchart` or `sequence` or `flowchart + sequence`.

If the update tool returns an error, apply CRITICAL failure handling.
```
